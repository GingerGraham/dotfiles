# ssh role

## Purpose

Sets up the user SSH configuration layer: directory structure, the `Include`
directive that activates `config.d/`, known host keys for common Git forges,
and a hardened client defaults snippet.

This role runs on all profiles (workstation, server, minimal).

## What this role does

1. **Creates directories** — `~/.ssh` (0700), `~/.ssh/config.d` (0700),
   `~/.ssh/cm_sockets` (0700).
2. **Ensures the Include directive** is the first line of `~/.ssh/config`
   (`Include ~/.ssh/config.d/*.conf`). Idempotent — install.sh may have
   written it already during Phase 3; this task will not duplicate it.
3. **Adds known host keys** for Git forges (GitHub, GitLab, Bitbucket by
   default) by running `ssh-keyscan` at play time. This means the role
   automatically picks up rotated host keys without a repo update.
4. **Deploys `config.d/00-defaults.conf`** — hardened client settings that
   apply to all connections: keepalive, ControlMaster multiplexing, modern
   key algorithm preferences, and safe defaults for ForwardAgent/X11.

## What this role does NOT do

- Generate SSH keys (personal or deploy). Personal key generation is the
  user's responsibility. Deploy keys for private companion repos are
  generated by `install.sh` Phase 3.
- Touch `~/.ssh/config.d/10-dotfiles.conf`. That file is owned by
  `install.sh` and holds the SSH host aliases for deploy keys. This role
  will not overwrite or remove it.
- Install or configure `ssh-agent` or `ssh-add` automation. That lives in
  `shell/config/core/ssh.sh` (shell layer).

## config.d load order

Files in `config.d/` are loaded in filename sort order. This role reserves
the `00-` prefix for the defaults snippet. install.sh uses `10-`. Leave gaps
for future additions:

| File | Owner | Purpose |
|---|---|---|
| `00-defaults.conf` | Ansible ssh role | Global `Host *` hardened defaults |
| `10-dotfiles.conf` | `install.sh` | Deploy key host aliases for companion repos |
| `20-*.conf` and beyond | User / future roles | Per-host or per-employer overrides |

## Variables

| Variable | Default | Override in | Purpose |
|---|---|---|---|
| `ssh_dir` | `~/.ssh` | `defaults/main.yml` | Base SSH directory |
| `ssh_config_d_dir` | `~/.ssh/config.d` | `defaults/main.yml` | Config fragment directory |
| `ssh_config_path` | `~/.ssh/config` | `defaults/main.yml` | Main SSH config file |
| `ssh_known_hosts_path` | `~/.ssh/known_hosts` | `defaults/main.yml` | known_hosts file |
| `ssh_cm_sockets_dir` | `~/.ssh/cm_sockets` | `defaults/main.yml` | ControlMaster socket directory |
| `ssh_known_hosts_forges` | `[github.com, gitlab.com, bitbucket.org]` | `host_vars/localhost.yml` | Forges to pre-trust in known_hosts |
| `ssh_keyscan_timeout` | `10` | `defaults/main.yml` | Timeout per ssh-keyscan call (seconds) |
| `ssh_deploy_client_defaults` | `true` | `host_vars/localhost.yml` | Deploy the `00-defaults.conf` snippet |

## Adding a private forge

If you have an internal GitLab instance or other private forge, add it in
`host_vars/localhost.yml`:

```yaml
ssh_known_hosts_forges:
  - github.com
  - gitlab.com
  - bitbucket.org
  - git.internal.example.com
```

## Extending client defaults

The `00-defaults.conf` template is Ansible-managed and will be overwritten on
each run. To add per-host overrides, create a `config.d/20-personal.conf` (or
any `2x-` name) on the machine directly. It will not be touched by this role.

For settings that should be version-controlled, add them as a template to this
role and deploy from a new task file.

## Dependencies

Requires the `common` role to have run first (sets `ansible_facts` used in
path variables).

## Running this role alone

```bash
ansible-playbook ansible/site.yml --tags common,ssh
```
